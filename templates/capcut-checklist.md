# Template — J. CapCut Editing Checklist

Checklist praktis untuk dieksekusi manual di **CapCut Mobile / Desktop**
berdasarkan plan A–I.

## Format

```
### CapCut Editing Checklist

#### 1. Project Setup
- [ ] New project, ratio 9:16 (1080×1920).
- [ ] Frame rate: 30fps (atau 60fps kalau sumber 60).
- [ ] Background: black solid (untuk hold tail di akhir).

#### 2. Opening Frame (0–2s)
- [ ] Visual: <visual function dari B01>
- [ ] Cut style: <hard cut from black / freeze>
- [ ] Subtitle pertama (S01): <isi>
- [ ] Optional: zoom-in ringan (1.0x → 1.05x) untuk emphasis

#### 3. Subtitle Style
- [ ] Font: <Bebas Neue / Komika Axis / Anton>
- [ ] Size: 80–100 (mobile readable)
- [ ] Color: white, stroke hitam 4–6px
- [ ] Position: 1/3 atas atau tengah, hindari tutupi wajah
- [ ] Highlight keyword: warna kontras (yellow #FFD60A atau red #FF453A)
- [ ] Animation: bounce-in atau fade-in cepat (≤0.15s)

#### 4. Zoom / Crop (9:16 dari source 16:9)
- [ ] Crop subjek utama ke center vertikal
- [ ] Reframe per beat — subjek pivotal di 1/3 atas
- [ ] Pan keyframe untuk wide-shot menjadi vertical-readable
- [ ] Hindari crop yang memotong wajah penting

#### 5. SFX Cues
- [ ] B01 cold open: <SFX impact / heartbeat>
- [ ] Bxx twist beat: <SFX whoosh + low boom>
- [ ] Bxx payoff: <silence drop + ambient pad>
- [ ] Subtitle pop: <click / tick subtle>

#### 6. Music / Pacing
- [ ] Track style: <suspenseful piano / ambient tension>
- [ ] BPM: 70–90 untuk drama, 100–120 untuk action.
- [ ] Volume: -18dB di bawah VO.
- [ ] Sidechain duck saat VO aktif.

#### 7. Cut Timing
- [ ] Patuhi pacing note dari Visual Beat Map (E).
- [ ] Hindari cut > 4 detik (kecuali di payoff window).
- [ ] Variasi: hard cut, cross-fade, freeze, slow-mo seperti di beat map.

#### 8. Ending Frame (~last 3s)
- [ ] Visual: <visual function dari beat terakhir>
- [ ] Cut style: <cross-fade to black / freeze + fade>
- [ ] Subtitle terakhir: <isi>
- [ ] Optional: tease text untuk Part 2 ("Part 2 →") di pojok atas

#### 9. Pre-Export QC
- [ ] Cek 0–3s di mute — apakah visual masih hook tanpa audio?
- [ ] Cek pacing — ada gap > 4s tanpa beat baru?
- [ ] Cek subtitle — semua chunk ≤ 5 kata, readable di hp?
- [ ] Cek volume balance — VO clear, music tidak overpower.
- [ ] Cek total durasi: 80–110s.

#### 10. Export
- [ ] Resolution: 1080×1920.
- [ ] Frame rate: sesuai project.
- [ ] Format: MP4, H.264.
- [ ] Bitrate: high (CapCut default sudah cukup untuk Shorts/TikTok).
- [ ] Watermark CapCut: hapus (di setting export).
```

## Aturan

1. Checklist harus **disesuaikan dengan plan**. Isi placeholder
   `<...>` dengan referensi spesifik dari Visual Beat Map (E),
   Subtitle Plan (G), dan Story Arc (C).
2. **Jangan default emoji** di subtitle/title.
3. **Jangan watermark CapCut** di final export.
4. **Audio**: VO harus dominan, music duck saat VO aktif.

## Anti-pattern

- Subtitle 8–10 kata per chunk — tidak readable di hp.
- Music sama volume dengan VO — VO ketelan.
- Cold open tanpa SFX — kurang punchy.
- Export ratio 16:9 — salah platform.

## Tips

- Kalau editing di mobile (CapCut iOS/Android), pakai **Auto Captions**
  dulu, lalu manual replace dengan Subtitle Plan (G). Auto Captions
  mempercepat timing, tapi text harus disesuaikan dengan VO.
- **Save preset** font + warna + stroke setelah project pertama —
  reuse untuk semua short.
- Render trial 720p dulu sebelum 1080p untuk cek pacing — hemat waktu.
